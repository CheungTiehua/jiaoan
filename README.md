# LeKai教案系统（K9-AI版）产品白皮书

> 版本：v1.0 | 日期：2026年6月 | 福建宁德

---

## 一、产品定义

### 1.1 这是什么

LeKai教案系统（K9-AI版）是一套**AI驱动的备课决策辅助系统**，面向小学语文教师，基于统编版（部编版）教材体系。

不是替代教师备课。是让教师看清"好教案长什么样、为什么好、怎么做出来"。

### 1.2 核心价值主张

| 传统备课 | LeKai |
|---------|-------|
| 老师自己找教案，质量参差不齐 | 内置知识库，收录优秀教案，AI 对比分析 |
| 看到教案不知道怎么化用 | 辅导说明拆解设计意图，"鱼渔同授" |
| 教研组长逐个审教案，费时费力 | 在线审核流程，段落标注，校本存档 |
| 新老师成长慢，老教师经验难传承 | 结构化同行参考，每条标注出处和可信度 |
| 教案散落在各台电脑上 | 统一平台，生成历史，一键导出 MD/DOCX |

### 1.3 命名含义

**K9** — 覆盖小学全学段（1-6年级），为未来扩展至初中预留空间。
**AI版** — 以 DeepSeek 大模型为核心引擎，区别于传统教案资源库。

## 二、功能架构

### 2.1 老师端

```
教材目录树（1-6年级全）
  │
  ├── 选课生成教案
  │     ├── 考点分析（这篇课文考试怎么考）
  │     ├── 结构化同行参考（名校/名师怎么教，每条标注出处+可信度）
  │     ├── 9章节标准教案（教材分析→学情→目标→重难点→过程→板书→作业→反思）
  │     └── 教案辅导说明（为什么这么设计，好在哪里）
  │
  ├── 对话修改教案（多轮修改，模糊请求自动追问）
  │
  ├── 单元整体规划（语文要素分解、课时分配、课型搭配）
  │
  ├── 课后反思引导（目标达成/过程复盘/学生反应）
  │
  ├── 教案评价（上传自己的教案，AI对照知识库评价+改进建议）
  │
  ├── 教案导出（MD / DOCX 一键下载）
  │
  └── 教学辅助
        ├── 扫码教案评分（1-5星，反哺检索质量）
        ├── 教研协作（建组/共享教案/分配备课任务/评论）
        ├── 生成历史（个人空间，追溯所有生成记录）
        └── 提交审核（进入校本审核流程）
```

### 2.2 管理端（校长/教研组长）

| 面板 | 功能 |
|------|------|
| 📊 仪表盘 | 全校教师数、教案生成数、审核通过数、年级覆盖热力图、教师活跃度表 |
| ✅ 审核队列 | 教师提交的教案，一键通过/打回，附带段落级标注（praise/improve/note） |
| 📤 教案入库 | 上传 .md/.docx/.txt 教案文档，自动格式化并入库到知识库 |
| 📦 知识库Chunk | 浏览/搜索/删除知识库中的单个知识点片段 |
| 🔧 提示词调优 | 在线编辑系统提示词，保存立即生效（无需重启） |
| 👥 角色管理 | 设置教师角色：teacher / reviewer / admin |
| ⭐ 反馈统计 | 教案评分分布、平均评分、高频标签 |
| 🔐 设备信息 | MAC地址、磁盘使用率、授权状态、系统版本 |
| 🩺 系统健康 | 磁盘/向量库/用户数据状态检查 |
| 💾 备份恢复 | 一键下载备份 zip（API Key脱敏），上传恢复 |

### 2.3 首次启动向导

盒子首次开机，浏览器打开自动进入三步向导：

1. **📡 WiFi设置** — 扫描WiFi列表 → 选择网络 → 输入密码 → 连接（可跳过，插网线）
2. **🔐 管理员密码** — 设置 admin 账号密码
3. **🔑 API Key** — 填写学校自己的 DeepSeek API Key

完成后跳转登录页。

## 三、技术架构

### 3.1 总体架构

```
浏览器 ──→ Nginx :80 ──→ /      → Next.js :3000 (前端)
                         → /api/  → FastAPI :8000 (后端)
                                     └── ChromaDB (向量库，嵌入式)
                                     └── DeepSeek API (云端，仅传文本)
```

### 3.2 技术栈

| 层 | 技术方案 | 说明 |
|----|---------|------|
| 后端框架 | FastAPI (Python 3.12+) | 异步高性能，自动生成 API 文档 |
| 前端框架 | Next.js 16 (React 19) + TailwindCSS | SSR友好，Markdown渲染 |
| 向量数据库 | ChromaDB (嵌入式 PersistentClient) | 零运维，SQLite 存储 |
| Embedding 模型 | BAAI/bge-small-zh-v1.5 (512维) | 96MB，纯中文优化 |
| 中文分词 | jieba | BM25 检索分词 |
| 大语言模型 | DeepSeek (deepseek-chat) | 支持多 Key 轮询 |
| 反向代理 | Nginx | 安全头 + API 限流 + 前端代理 |
| 进程管理 | systemd (盒子) / Docker Compose (云) | 开机自启，崩溃自动重启 |

### 3.3 检索架构

```
用户输入"三年级 富饶的西沙群岛 修辞手法"
          │
          ▼
    ┌─────────────────────────────────────┐
    │         混合检索 (Hybrid)            │
    │                                     │
    │  BM25 (jieba分词)  +  ChromaDB向量   │
    │         │                  │         │
    │         └──── RRF融合 ─────┘         │
    │                  │                  │
    │          Top-K 检索结果              │
    └─────────────────────────────────────┘
          │
          ▼
    Evidence Pack 教学维度分层
    ├── 教材分析参考
    ├── 教学目标参考
    ├── 教学过程参考
    ├── 板书设计参考
    ├── 作业布置参考
    └── 教学反思参考
          │
          ▼
    ┌──────────────────────────────────────┐
    │        DeepSeek 四层生成              │
    │                                      │
    │  Step 1+2 并行: 考点分析 + 同行参考    │
    │  Step 3: 教案生成 (注入1+2)           │
    │  Step 4: 辅导说明 (注入1+2+3)         │
    └──────────────────────────────────────┘
```

### 3.4 安全体系

| 维度 | 措施 |
|------|------|
| 密码存储 | PBKDF2-SHA256，100,000次迭代，随机盐 |
| 认证鉴权 | Bearer Token，7天有效期，服务重启不丢失 |
| 角色权限 | teacher / reviewer / admin 三级 |
| 速率限制 | 登录10次/分钟，注册5次/5分钟，生成20次/分钟，线程安全 |
| 数据保护 | 原子写入(tmp+rename)，文件锁(fcntl)防并发 |
| 路径安全 | os.path.basename 防目录穿越，resolve() 防 Zip Slip |
| 备份安全 | Zip Bomb 三层防护(大小/压缩比/总量) + API Key 脱敏 |
| 传输安全 | Nginx 安全头(X-Content-Type/X-Frame/XSS-Protection) + 限流30r/s |
| 系统隔离 | 非root 用户运行，ProtectSystem=strict 沙箱 |

## 四、部署方案

### 4.1 盒子部署（推荐）

硬件：N150 处理器 / 12GB DDR4 / 256GB NVMe SSD + 256GB SATA SSD（备份盘）

```
一台小电脑，交钥匙交付：
  ├── Ubuntu Server 22.04 LTS 预装
  ├── 应用已编译部署（Nuitka 编译后端为二进制 + Next.js standalone 前端）
  ├── MAC 地址硬件授权绑定
  ├── 系统级开机自启 (systemd)
  ├── 自动定时备份（cron，每天凌晨3点→备份SSD，保留7天）
  └── 一键恢复（换新电脑 → 插入备份SSD → bash restore_backup.sh）
```

### 4.2 Docker 部署（云服务器）

```bash
# .env 文件配置 DeepSeek API Key
echo "DEEPSEEK_API_KEY=sk-xxx" > .env

# 启动
docker compose up -d

# 访问 http://服务器IP （Nginx :80 代理前端+后端）
```

Docker Compose 三容器：Nginx + Next.js + FastAPI，健康检查 + 资源限制 + 日志轮转。

### 4.3 盒子运维

| 操作 | 命令 |
|------|------|
| 查看状态 | `systemctl status lekai-backend lekai-frontend nginx` |
| 查看日志 | `journalctl -u lekai-backend -f` |
| 重启服务 | `systemctl restart lekai-backend lekai-frontend` |
| 手动备份 | 管理端 → 💾 备份恢复 → 下载备份 |
| 更新系统 | `scp lekai_update.tar.gz root@盒子IP:/tmp/ && ssh root@盒子IP "bash box_update.sh"` |
| 灾难恢复 | 插入备份SSD → `bash restore_backup.sh` |

## 五、用户角色与工作流

### 5.1 角色定义

| 角色 | 权限 |
|------|------|
| **教师 (teacher)** | 选课生成教案、对话修改、教案评价、教研协作、个人历史、导出教案 |
| **教研组长 (reviewer)** | 教师全部权限 + 审核教案+段落标注、查看仪表盘、教案入库 |
| **管理员 (admin)** | 教研组长全部权限 + 角色管理、提示词调优、备份恢复、设备管理 |

### 5.2 典型工作流

**日常备课流程：**

```
教师打开浏览器 → 登录 → 左侧目录树选课
  → 点击"生成" → 查看考点分析 → 参考同行做法
  → 获得完整教案 → 对话调整细节 → 满意后导出/打印
  → 点击⭐评分反馈 → 提交教研组长审核
```

**教研组集体备课流程：**

```
教研组长创建教研组 → 邀请成员加入 → 分配备课任务
  → 各成员生成初稿 → 共享到教研组 → 互相评论
  → 教研组长审核通过 → 存档为校本教案
```

**新教师成长流程：**

```
新教师生成教案 → 查看辅导说明（理解设计意图）
  → 上传自己写的教案 → AI对照优秀教案评价
  → 反复练习 → 教案修改次数递减 → 专业成长可量化
```

**校长管理流程：**

```
校长打开 /admin → 仪表盘扫一眼全校数据
  → 查看哪些老师活跃、哪些年级已覆盖
  → 评审期间导出全校教案数据
```

## 六、知识库管理

### 6.1 知识库结构

```
knowledge-base/
├── grade-1/  ← 空，待学校入库
├── grade-2/  ← 空，待学校入库
├── grade-3/
├── grade-4/
├── grade-5/
└── grade-6/
```

### 6.2 入库方式

管理端 → 📤 教案入库 → 上传 .md / .docx / .txt 教案文档 → 系统自动 DeepSeek 格式化 → 入库到 ChromaDB 向量库 → 立即可检索。

权限：仅管理员和教研组长可入库。保证知识库质量可控。

### 6.3 教案来源建议

- 国家中小学智慧教育平台（basic.smartedu.cn）
- 教研员/名师公开课教案
- 学校历年优秀校本教案
- 学科网、百度文库中筛选的优质教案

## 七、产品边界（不做的事）

| 不做 | 原因 |
|------|------|
| 自动生成 PPT/课件 | 专注教学设计的思考层，课件制作交给专业工具 |
| 学生端/作业批改 | 聚焦教师备课场景，不扩展到教学全过程 |
| 多学科支持 | 当前仅小学语文，深度优先于广度 |
| 公有云 SaaS | 学校数据不出校，盒子私有部署 |
| 移动 App | 浏览器 Web 够用，App 是维护黑洞 |
| 接第三方 LMS | 不是平台型产品，保持独立简洁 |
